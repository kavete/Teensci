Javascript in HTML

This article is going to talk about javascript briefly and then we will see how to use javascript in HTML.

Overview of Javascript

Javascript is one of the most popular programming languages. It is majorly used to add interactivity to webpages. It is used in conjunction with HTML (Hypertext Markup Language) and CSS (Cascading Style Sheets) for front end web development. ECMAScript is the official name for Javascript.

Today, Javascript is the most used programming language to develop websites, with close to 95% of websites using it. There are also several Javascript libraries which make Javascript programming simpler. One good example of such a library is jQuery.

If you plan on practicing javascript programming, all you need is a browser -preferably Chrome or Safari, and a text editor. For PC (Windows, Mac and Linux) I recommend Brackets, VScode or Atom as the editor and if you are on android you could use Acode, Quoda, anWriter or DroidEdit

To use or learn javascript effectively for web development, you might need a basic understanding of HTML and CSS

If you are looking for detailed tutorials on Javascript, I would recommend;

  • W3schools.com
  • Javascript.info
  • Goalkicker.com/JavaScriptBook

Case uses of Javascript

Generally Javascript is used in web development. It was initially used for front end web development but has developed into just more than that. Today, Javascript is used for server side programmimg due to the introduction of nodejs -A javascript runtime environment. Javascript is also used in Game programming in conjunction with HTML, majorly by using the <canvas> tag. React Native and Phonegap have also made it possible to develop mobile apps for both Android and iOS using javascript.

Including Javascript in HTML

Javascript is inserted in HTML by using the <script> tag. It can be on the head section or before the closing </body> tag. Most developers place javascript before the ending </body> tag to speed up page load. If you place javascript in the head section, your page might take a little more time to load. Place your javascript in the head section only if your page elements really need it for first impression.

Javascript in the head section


<!doctype html>
<html> 
   <head> 
       <script> 
           //Javascript code 
       </script> 
   </head> 
   <body> 
    <!--
        HTML elements 
    -->
   </body>
</html>

Javascript before the </body> tag


<!doctype html>
<html> 
   <head> 
    <!--
        Meta data
     -->
   </head> 
   <body> 
    <!--
        HTML elements 
     -->
      <script> 
           //Javascript code 
      </script> 
   </body>
</html>

External Javascript

Javascript can also be used in HTML by placing it in another file rather than writing it directly in HTML. You are going to find external javascript very useful especially when linking to large javascript files or javascript files not written by you. A good example is linking to a javascript library such as jQuery. External javascript can be linked in the head section or before the closing </body> tag. Javascript files have a file extension of .js

Suppose you have a javascript file called jscript.js placed in the same folder as the html file, you can link it by using the <script> with an src attribute pointing to your javascript file.

External Javascript in head section


<!doctype html>
<html> 
   <head> 
       <script src="jscript.js">    
       </script> 
   </head> 
   <body> 
    <!--
        HTML elements 
     -->
   </body>
</html>

External javascript before the </body> tag


<!doctype html>
<html> 
   <head> 
     <!--
        Meta data
     -->
   </head> 
   <body> 
    <!--
        HTML elements
     --> 
     <script src="jscript.js"> 
            
       </script> 
   </body>
</html>

As discussed earlier, placing javascript before the closing </body> tag speeds up page load.

Javascript Output

The document.write(), console.log() and innerHTML() methods are the main used to output content in HTML.

The document.write() method should only be used for testing purposes. It overwrites all the existing HTML elements.


<!doctype html>
<html>
   <head>
      <script>
        document.write("<h1>Hello world</h1>");
      </script>
   </head>
   <body>
  
   </body>
<html>

The above code outputs a level one heading. The text inside the document.write() can contain markup tags as in the example above.

The console.log() method is used to output content in the browsers console. It is meant for debugging purposes. If you are on windows and you are using Microsoft Edge or Internet Explorer, you can press F12 to access the console. For Chrome you can use the shortcut Shift + CTRL + J.


<!doctype html>
<html>
   <head>
      <script>
        console.log("Hello   world");
      </script>
   </head>
   <body>
  
   </body>
<html>

The above code prints Hello World in the browser's console.

The innerHTML() method can also be used to output content in HTML. To use this method, you need to have selected an HTML element (You should be able to access the DOM). There are several ways of accessing HTML elements. In the example below we have used document.getElementById().


<!doctype html>
<html>
 <head>
  <!--
    Meta data
   -->
 </head>
 <body>
  <h1>Example</h1>
  <div id="demo"> </div>
 <script>
  var elem = document.getElementById("demo");
  elem.innerHTML("<p>This is new text from javascript</p>");
 </script>
 </body>
</html>

The above code will output;

Example

This is new text from Javascript

The <noscript> tag

The content between these tags is shown when the clients browser doesn't support Javascript or they have disabled it. You can place the <noscript> tags in the head section as shown in the code below.


<!doctype html>
<html>
 <head>
  <noscript>
   Your browser does not support  Javascript or you have disabled it.
   Javascript is needed to continue.
  <noscript>
  <script>
   //Javascript code
  </script>
 </head>
 <body>
  <!--
    HTML elements
   -->
 </body>
</html>

Javascript comments

Comments are ignored by the browser. Comments make your code readable to yourself and others. They also help in understanding the coder's thought process. Comments can be used for debugging purposes since commenting a certain javascript code prevents it from being executed.

There are Single line comments and multi-line comments. Single line comments should start with two forward slashes ( // ). Comments spanning more than one line should start with a forward slash and an asterisk ( /* ) and end with an asterik and a forward slash ( */ ). Check out the comments below.


<script>

// single line comment

/*

Multi line 
comment 

*/
</script>
